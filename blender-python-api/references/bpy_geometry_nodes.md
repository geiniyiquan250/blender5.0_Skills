## 核心概念

### 字段（Fields）

字段是一个函数：一组指令可以将任意数量的输入转换为单个输出。字段的结果可以根据不同的输入数据计算多次。字段用于几何节点的各个方面，允许进行对每个元素（网格顶点、面等）有不同结果的计算。

```python
# 字段输入示例
# Set Position 节点的输入字段依赖 Position 和 Index
# 将它们转换为向量，使用单条指令
```

**字段可视化**

插座形状用于表示哪些插座是字段，哪些是常规数据。有三种可能的插座形状，每种都可视化其"字段状态"：

- **圆形**：插座需要单个值，不能接受字段输入。对于输出插座，表示节点始终输出单个值。
- **菱形**：插座可以接受字段输入，或输出字段。常量单个值可以连接到这些插座，但输出通常不会随元素变化。
- **带点的菱形**：插座可以是字段，但目前是单个值。这有助于跟踪单个值的计算位置，而不是具有许多不同结果的字段。

当两个支持字段的节点插座之间建立连接时，节点连接将绘制为虚线。如果错误地将非字段插座连接到字段插座，连接将绘制为实心红线表示错误。

**字段上下文**

所有字段节点都在与其连接的数据流节点的上下文中工作。上下文通常由几何组件类型和属性域组成，这决定了从输入节点检索哪些数据。

```python
# 字段评估示例
# Set Position 节点的输入字段被评估一次
# 节点向后遍历以从字段输入节点检索输入

# 当添加第二个 Set Position 节点时
# 相同的字段节点树被评估两次，每个数据流节点一次
# 在第二个 Set Position 节点，结果将不同
# 因为其几何输入将已经具有来自第一个节点的更改位置

# 使用 Capture Attribute 节点存储初始位置的副本
# 字段输入节点的评估是完全不同的步骤
# 稍后 Set Position 节点的输入字段不使用实际位置
# 而是使用匿名属性副本
```

### 几何实例（Instances）

除了存储真实数据（如网格或曲线）外，对象还可以存储实例，这些实例可以引用更多几何体、对象或集合。实例化的目的是允许复制几何体并将其存储在对象中，而无需复制实际数据。这种优化允许渲染引擎（如 Cycles）更好地处理许多不同位置的相同几何体数据。

每个实例跟踪其对应的几何体以及与源几何体相比的实例化变换方式。实例还可以存储 `id` 属性，用于在实例移动时校正运动模糊。

```python
# 实例创建示例
# 使用 Instance on Points 节点创建实例
# 实例可以存储几何体，几何体可以包含实例
# 因此可以创建嵌套实例化
```

**嵌套实例化**

由于实例可以存储几何体，而几何体可以包含实例，因此可以进行嵌套实例化。默认情况下，Instance on Points 节点将通过在实几何体的点上实例化实例来创建嵌套实例。

**实例处理**

几乎所有处理几何体的节点都是通过单独处理每个唯一几何体来处理，而不是已实现的几何体。例如，如果在示例末尾放置 Subdivision Surface 节点，它只需要细分三个网格，而不是每个网格实例。这种方法可以大大提高性能，但这意味着操作的结果对于每个特定几何体的所有实例都是相同的。

```python
# 为了对每个实例有唯一结果
# 使用 Realize Instances 节点
# realize_instances 节点将实例转换为唯一几何体
# 实现后会占用更多内存，操纵几何体必须逐个处理
```

### 几何域（Domains）

几何体由不同的组件类型组成：网格、曲线、点、体积和实例。每个组件类型进一步分为不同的域：

- **点域**：网格顶点、曲线控制点和曲线点
- **边域**：网格边
- **面域**：网格面
- **曲线域**：样条线
- **面角域**：网格面角
- **实例域**：实例

### 属性（Attributes）

属性是可以附加到几何体不同域的数据值。属性可以是内置的（如位置、法线）或用户定义的。属性在几何节点中通过属性名引用。

```python
# 属性操作
# Store Named Attribute 节点存储属性
# Named Attribute 节点读取属性
# Attribute Statistic 节点计算统计信息
# Remove Named Attribute 节点删除属性
```

### 检查与调试（Inspection）

检查几何节点树中的中间值在构建/理解节点树时或试图找出某些东西不工作的原因时非常有用。Blender 提供了多种工具来理解节点树的工作方式或为什么不工作。

**插座检查**

插座检查显示最后一次评估期间插座中的值信息。对于原始数据类型（如整数、向量和字符串），显示实际值。对于几何体插座，只存储有关几何体的一些数据，包括几何体包含的数据类型集及其元素计数。

**属性搜索**

点击修饰符中的属性输入时显示属性搜索。它包含在修饰符或节点执行中该点可用的所有属性列表。

**节点警告**

当节点的输入无效时，标题中会显示警告。将鼠标悬停在警告图标上会显示错误消息。这些警告仅在执行节点时生成，因此节点必须连接到 Group Output 才能有警告。

**节点计时叠加**

节点计时显示节点执行节点组上一次评估所需的时间。节点计时叠加可以在节点编辑器右上方的叠加弹出窗口中打开。当节点组在多个地方使用时，计时取决于节点编辑器的上下文。

## 模拟区域

模拟区域允许前一帧的结果影响下一帧。通过这种方式，即使是简单规则的集合也可以随着时间的推移产生复杂的结果。最常见的是物理模拟，具有针对物理现象的特定求解器。

添加模拟时，会添加两个节点，在它们之间定义一个"模拟区域"。连接到 Simulation Input 节点的输入仅在模拟开始时评估一次，传递给下一个模拟状态并最终输出。其他节点可以从外部链接到模拟区域内。这些节点会根据给定帧的值在每一步重新评估。

无法有任何链接指向外部。模拟的结果只能通过 Simulation Output 节点访问。这也允许子帧插值以实现运动模糊。

```python
# 模拟区域节点
# Simulation Input 节点 - 提供初始状态
# Simulation Output 节点 - 输出最终结果

# 模拟输入类型
# Geometry - 标准几何体输入
# Delta Time - 帧之间的时间间隔（秒）
# Skip - 跳过模拟区域

# 匿名属性不会由模拟节点传播
# 除非它们显式存储在模拟状态中
# 因为检测哪些匿名属性将需要模拟
# 需要查看未来以确定哪些数据是必要的
```

模拟与动画系统绑定，支持子步进。它仅在动画帧更改时评估，并像 Blender 中现有的物理模拟一样被缓存。

## bpy_geometry_nodes_zones_fields

# 几何节点区域与场模块

## 核心概念：场（Fields）

### 场的本质

从根本上说，场是一个函数：一组指令可以将任意数量的输入转换为单个输出。场的结果可以根据不同的输入数据计算多次。场用于几何节点的各个方面，允许进行对每个元素（网格顶点、面等）有不同结果的计算。

```python
# 场的工作原理
# 场是定义如何为每个元素计算值的指令集
# 例如，连接到 Set Position 节点的场
# 依赖于 Position 和 Index 两个输入
# 使用单条指令将它们转换为向量

# 场 vs 常量值
# 常量值：对于所有元素返回相同的值
# 场：对于每个元素可以返回不同的值
```

### 场的可视化

插座形状用于表示哪些插座是字段，哪些是常规数据。有三种可能的插座形状：

**圆形插座**
- 插座需要单个值，不能接受字段输入
- 对于输出插座，表示节点始终输出单个值
- 常量值使用此形状

**菱形插座**
- 插座可以接受字段输入，或输出字段
- 常量单个值可以连接到这些插座
- 但输出通常不会随元素变化
- 字段连接使用此形状

**带点的菱形插座**
- 插座可以是字段，但目前是单个值
- 有助于跟踪单个值的计算位置
- 而不是具有许多不同结果的字段
- 意味着插座位检查将显示值而非字段输入名称

```python
# 连接规则
# 当两个支持字段的节点插座之间建立连接时
# 节点连接将绘制为虚线

# 错误连接
# 如果错误地将非字段插座连接到字段插座
# 连接将绘制为实心红线表示错误

# 示例
# 每个点将向上移动5米 - 场对每个元素有相同的值
# 位置将加倍 - 场对每个元素有不同的值
# 因为偏移是每个点的位置
```

### 场节点类型

节点可以分为两类：传递几何体的数据流节点，和对数据按元素操作的场节点。

**数据流节点**
- 具有几何体输入和几何体输出的节点
- 实际上改变将从几何节点修饰符输出的几何体数据
- 处理几何体的主变换和操作

**函数节点**
- 具有菱形插座输入和输出的节点
- 类似于将由数据流节点评估的指令
- 示例：数学节点和更复杂的节点如 Geometry Proximity

**输入节点**
- 为字段评估过程提供数据
- 本身没有意义；必须在数据流节点（几何体）的上下文中评估才能实际输出值
- 示例：内置属性输入节点如 Position 和 ID
- 也包括选择节点如 Endpoint Selection

```python
# 字段输入来源
# 也可以来自其他处理几何体的节点
# 如 Distribute Points on Faces 节点
# 以匿名属性的形式提供
```

## 场的上下文

### 上下文概念

所有字段节点都在与其连接的数据流节点的上下文中工作。上下文通常由几何体组件类型和属性域组成，这决定了从输入节点检索哪些数据。

```python
# 上下文的重要性
# 同一个字段节点树在多个地方使用
# 不一定会输出相同的数据
# 因为字段节点树将为每个数据流节点评估
# 可能从不同或已更改的几何体检索数据

# 评估流程示例
# Set Position 节点的输入字段被评估一次
# 节点向后遍历以从字段输入节点检索输入

# 添加第二个 Set Position 节点时
# 相同的字段节点树被评估两次
# 每个数据流节点一次
# 在第二个 Set Position 节点
# 结果将不同
# 因为其几何体输入将已经具有来自第一个节点的更改位置
```

### 捕获属性解决上下文问题

通常需要在更改几何体后使用相同的字段值。Capture Attribute 节点评估字段，将结果复制到几何体上的匿名属性。

```python
# 使用 Capture Attribute
# 捕获初始位置的副本
# 评估 Capture Attribute 节点的字段输入是完全不同的步骤
# 稍后 Set Position 节点的输入字段
# 不使用实际位置
# 而是使用匿名属性副本

# 匿名属性的特点
# 仍然存储在几何体上
# 当几何体与其他节点更改时
# 它们会自动插值
# 只要节点链接仍然可访问
# 属性引用将可用
```

### 场的常见误解

```python
# 误解示例
# 相同的场在两个不同位置使用
# 认为会得到相同结果

# 实际情况
# 场在每个数据流节点重新评估
# 获取当前上下文的几何体数据
# 因此结果可能不同

# 解决方案
# 使用 Capture Attribute 捕获值
# 存储在几何体上供后续使用
# 避免重复评估导致的值差异
```

## 区域类型详解

### 模拟区域（Simulation Zone）

#### 模拟区域概述

模拟区域允许前一帧的结果影响下一帧。通过这种方式，即使是简单规则的集合也可以随着时间的推移产生复杂的结果。最常见的是物理模拟，具有针对物理现象的特定求解器。

```python
# 模拟区域用途
# 创建随时间演化的几何体
# 支持反馈循环
# 实现物理模拟效果
# 创建复杂的程序化动画

# 典型应用
# 粒子系统
# 流体模拟
# 布料效果
# 群体行为
```

#### 模拟区域节点结构

当添加模拟时，会添加两个节点，在它们之间定义一个"模拟区域"。

**Simulation Input 节点**
- 连接到 Simulation Input 节点的输入仅在模拟开始时评估一次
- 传递给下一个模拟状态并最终输出
- 这些是模拟的初始条件

**Simulation Output 节点**
- 无法有任何链接指向外部
- 模拟的结果只能通过 Simulation Output 节点访问
- 这也允许子帧插值以实现运动模糊

```python
# 输入类型
# Geometry - 标准几何体输入
# Delta Time - 帧之间的时间间隔（秒）
# Skip - 跳过模拟区域

# 外部链接节点
# 可以从外部链接到模拟区域
# 那些节点将根据给定帧的值
# 在每一步重新评估

# 匿名属性处理
# 匿名属性不会由模拟节点传播
# 除非它们显式存储在模拟状态中
# 因为检测哪些匿名属性将需要模拟
# 需要查看未来以确定哪些数据是必要的
```

#### 模拟时钟

模拟与动画系统绑定，支持子步进。它仅在动画帧更改时评估，并像 Blender 中现有的物理模拟一样被缓存。

```python
# Delta Time
# 帧之间的时间（秒）
# 本质上是渲染帧率的倒数
# 此增量用于通过连接到依赖于速率的节点设置来驱动模拟
# 这将确保模拟播放的一致性
# 即使帧率发生变化

# 评估时机
# 仅在动画帧更改时评估
# 缓存机制
# 像现有物理模拟一样缓存
```

#### 模拟烘焙

### 重复区域（Repeat Zone）

#### 重复区域概述

重复区域执行一组节点多次。这允许迭代算法和程序化生成。

```python
# 重复区域用途
# 实现迭代算法
# 程序化生成复杂几何体
# 分层构建结构
# 创建递归模式

# 典型应用
# 分形生成
# 迭代变形
# 累加效果
# 渐进式细化
```

#### 重复区域节点结构

该区域由左侧的输入节点、右侧的输出节点和中间放置要重复的节点的橙色区域组成。

**第一次执行时**
- 评估其输入
- 将它们转发到其内部节点
- 这些内部节点可以写入输出节点
- 为下一次迭代提供输入
- 并为最后一次迭代后的结果提供区域结果

**内部节点**
- 也可以从区域外部的节点获取输入
- 这些输入在每次迭代中相同
- 但是区域内的节点不能将其输出发送到区域外的节点

```python
# 输入配置
# Iterations - 执行次数
# Iteration 插座给出从0开始的当前迭代索引
# Geometry - 默认几何体输入
# 可以添加更多输入
# 通过将节点的输出连接到区域的空白输入
# 或使用节点属性面板中的重复项列表

# 输入管理
# 可以通过在区域本身中
# 或在重复项列表中
# Ctrl+左键单击来重命名输入

# 迭代访问
# Iteration 插座提供当前迭代编号
# 可用于根据迭代次数改变行为
```

#### 重复区域示例

```python
# 金字塔生成示例
# 生成具有可配置层数的金字塔
# 每次迭代创建一个立方体
# 将其带入位置
# 将其与前一次迭代的结果连接

# 迭代逻辑
# 迭代 0：创建第一个立方体
# 迭代 1：在第一个立方体上方创建第二个
# 迭代 2：继续堆叠
# ...
# 最终：连接所有层

# 使用场景
# 生成分形结构
# 创建递归模式
# 累加几何体变换
# 程序化建筑生成
```

### 循环几何区域（For Each Geometry Zone）

#### 循环几何区域概述

此区域类型允许对几何体的每个元素执行节点。例如，节点可以处理网格的每个面或每个实例。

```python
# 区域用途
# 为输入几何体的每个元素生成大型或复杂几何体的任务
# 例如，为每条输入曲线生成独特的树
# 或在每个输入面上生成独特的建筑

# 性能考虑
# 对于处理少量几何体不太合理
# 在那种情况下（例如分别处理角色的每根头发）
# 可能总是比处理较少较大几何体更慢
# 处理每个元素的额外灵活性
# 代价是 Blender 无法很好地优化操作
```

#### 循环几何区域输入输出

**输入**
- Geometry：迭代其元素的几何体
- Selection：要处理的所选域的子集
- Index：源几何体中元素的索引

**输出**
- 主几何体输出在主输出几何体上创建属性
- 区域内的每个单个值成为当前索引处的属性值
- 生成面板中的输出（包括默认几何体输出）
- 从每个元素生成的几何体连接在一起

```python
# 域选择
# Point - 点域
# Edge - 边域
# Face - 面域
# Corner - 角域
# Spline - 样条域
# Instance - 实例域

# 元素输出
# 输入几何体被拆分为每个元素的几何体
# 这是当前迭代的单个元素几何体
# 面角域不可用
# 因为面角不能没有其面而存在

# 性能优化
# 将大型几何体拆分为许多小元素可能效率低下
# 因为如果不在节点图中使用
# 此输出不会被计算
# 不使用它通常会提高性能
```

```python

# 避免过度指定精度
```
